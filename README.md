# ATM Simulation in C

A console-based ATM simulation project developed in **C** during my university studies. This project was built as a learning-focused application to better understand the core logic behind ATM systems and strengthen my programming fundamentals through hands-on practice.

## Overview

The aim of this project is to simulate basic ATM operations in a console environment. It focuses on core banking actions, user interaction, and simple data management using file operations.

This project was developed as an academic exercise and reflects my early practical experience in software development with the C programming language.

## Objectives

- Understand the basic working logic of ATM systems
- Practice procedural programming in C
- Improve problem-solving and algorithmic thinking
- Gain experience with file handling and data persistence
- Build a menu-driven console application

## Features

The application includes several basic ATM-related operations, such as:

- Creating and managing customer records
- Viewing customer/account information
- Balance inquiry
- Cash deposit
- Cash withdrawal
- Basic transaction processing
- File-based data storage using `.dat` files
- Console-based menu navigation and user interaction

## Technologies Used

- **Language:** C
- **Concepts:** Procedural Programming, Functions, Structures, File Handling
- **Storage:** File-based persistence with `.dat` files

## What I Learned

Through this project, I gained practical experience in:

- Writing structured programs in C
- Designing menu-driven console applications
- Using functions to organize program logic
- Working with structures to represent data
- Handling file read/write operations
- Managing user input and program flow
- Improving debugging and logical thinking skills

## Project Structure

This repository currently contains the main source file of the project:

- `main.c` — main implementation of the ATM simulation

## How to Run

### Compile

```bash
gcc main.c -o atm
```

### Run

```bash
./atm
```

> On some systems, the executable name may differ depending on the compiler and operating system.

## Notes

- This is a **learning-oriented academic project**, not a production-ready banking application.
- The project was created to practice programming concepts and understand system behavior in a simplified way.
- Security, scalability, and production-level architecture were not the primary goals of this work.
